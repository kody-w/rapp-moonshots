Adaptive Orb

An eyes-up operating layer for AI when hands and attention are occupied. Start without sensors, then optionally add voice and front-camera gestures one permission at a time.

Before permission: camera frames are analyzed only in ephemeral local memory and are never recorded. The Web Speech API is a browser service and may send audio to a browser or OS vendor for processing. This app cannot audit or control that vendor processing. The useful default requests neither camera nor microphone. Microphone and camera are separate later choices, never prerequisites.         Privacy, sensing, and recovery boundary

        Ephemeral frames: a 48×36 analysis canvas is cleared every processing turn. Raw pixel bytes are zeroed. One rolling derived grayscale comparison and at most one tracked detector working copy may exist; each is zeroed on replacement, completion, or shutdown.

        Honest estimate: FaceDetector supplies a transient face box/head-position proxy when available. Otherwise local frame motion is used. Neither path is precise gaze or identity inference.

        Freshness: new media frames, valid changing image content, and processed estimates expire independently. Any stale signal clears the arm and freezes sensor-derived confirmation.

        Speech and AI boundary: Web Speech may use browser/vendor processing. Conversation text stays in memory. Demo AI makes no request; companion mode sends only the explicit Brainstem contract to same-origin /api/chat. Exports omit text.

        Install, offline behavior, and iOS help

        iOS: Safari Share → Add to Home Screen. Installability and offline use do not guarantee live camera, microphone, or speech inside standalone mode. The app capability-detects each launch; webkitSpeechRecognition is used only when present. Sensor-free parity always remains available.

        Install tutorial: (1) open this HTTPS page in Safari, (2) tap Share, (3) choose Add to Home Screen, and (4) open Adaptive Orb from the new icon. Start sensor-free; add microphone and camera only if that standalone runtime offers them.

        Standalone degradation: if an installed runtime lacks an API or permission, choose sensor-free mode. On iOS, use Open in Safari for live sensors where the same HTTPS URL can be opened externally; Safari performs a separate permission check and availability is still not guaranteed.

        Offline: the service worker caches only the app shell, manifest, and local icons. It never caches media, conversation, companion responses, calibration, or metrics.
