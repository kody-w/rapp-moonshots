# Adaptive Orb pitch

The tournament did not need a fourth prototype. It needed one product that knew
when each finalist was right.

Adaptive Orb turns modality into an adaptive grammar, not a page boundary:

- Orbit handles the ambiguity of “what do you want?”
- Compass handles the precision of “which one?”
- Tunnel handles the memory burden of “where am I, and how do I get back?”

The center never moves. The route never changes owners. Task values, undo
history, sensor freshness, independent freeze causes, and metrics survive every
transition. Spoken mode names allow agency without splitting the experience.

Its strongest feature is what it refuses to do. Gaze cannot execute. Stale
frames cannot confirm. Center cancels. Stop/cancel/undo preempt. A deliberate
wrong branch is recoverable. Final confirmation sends nothing outside the
browser. Sensor-free parity keeps the exact task available.

The 8.7-second deterministic fixture visibly uses all three modes, injects and
recovers from sensor loss, blocks a gaze commit, undoes a wrong tunnel, and
finishes the exact cobalt-beacon task with zero false commits. The generated
artifact is one dependency-free HTML file with Clawpilot theming, local-only
sensing, no persistence, and no external assets.

Adaptive Orb is not claiming perfect gaze, universal accessibility, or measured
human speed. It is making a narrower, stronger claim: adaptive interaction can
change shape without changing safety state.
