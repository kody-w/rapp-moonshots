# Tournament verdict — Adaptive Orb

No single prototype wins Moonshot 003.

The result is **Adaptive Orb**, one mobile-first AI application that uses each
interaction grammar for the situation where it is strongest:

- **Voice Orbit** captures broad intent and predicts next actions.
- **Gaze Compass** handles precise selection among stable options.
- **Gesture Tunnel** handles nested explanations, revisions, tools, and return paths.

The modes share one AI conversation, task state, undo history, safety state,
sensor lifecycle, and metrics model. Users can switch by voice, while the app
also chooses a mode from the shape and depth of the current AI response.

## Why this is the moonshot

Phones made AI portable but still make people stop, look down, and type.
Adaptive Orb explores an eyes-up interaction layer for kitchens, workshops,
field work, quick note capture, and accessibility—situations where hands or
attention are occupied.

It is a moonshot because the long-term category is larger than a radial menu:
an adaptive spatial language for speaking with AI in the physical world.

## What exists now

- one installable mobile PWA;
- offline deterministic demo AI;
- optional server-side RAPP brainstem connector;
- one permission ceremony with progressive microphone/camera grants;
- sensor-free, touch, keyboard, and switch parity;
- contextual and spoken mode switching;
- shared memory-only conversation;
- deterministic multi-turn simulation using every mode;
- iOS Add-to-Home-Screen guidance and runtime capability degradation;
- 85 Node tests, 8 Python tests, and 29 policy/evidence checks.

## Honest boundary

The prototype proves the unified application, safety state machine, local
privacy controls, PWA packaging, deterministic conversation flow, and
integration contract.

It does **not** yet prove webcam gaze accuracy, fatigue, speed advantage, or
accessibility benefit with real users. Those require the next tournament:
human trials across lighting, movement, speech capability, and iOS devices.
